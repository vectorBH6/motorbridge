# Release Test Notes - 0.3.5

Date: 2026-05-20

## Scope

This release focuses on ABI/thread-safety hardening, Python CLI maintainability
without public entrypoint breakage, CoreController polling lifecycle cleanup,
RobStride scan efficiency, strict CI alignment, and version/documentation
alignment.

## Key Changes

- Public Python binding methods and CLI entrypoints remain compatible:
  `motorbridge.cli:main`, `python -m motorbridge.cli`, `python -m motorbridge`,
  `motorbridge-cli`, and legacy flat run arguments still work.
- C ABI function names and signatures remain stable. Internally, controller and
  motor handles now serialize same-handle calls with a mutex.
- `CoreController` now stops its background polling worker on `Drop`.
- Python RobStride scan now opens one controller per `feedback_id` candidate
  instead of reopening the CAN socket for every `(motor_id, feedback_id)` pair.
- `open_can_bus()` is the preferred cross-platform classic-CAN backend selector;
  `open_socketcan()` remains available as a compatibility alias.
- `releases/` artifacts are ignored and no longer tracked by Git.

## Compatibility Checks

Python CLI package split compatibility:

```bash
PYTHONPATH=bindings/python/src python - <<'PY'
from motorbridge.cli import main, _scan_command
print(callable(main), callable(_scan_command))
PY
PYTHONPATH=bindings/python/src python -m motorbridge.cli --help
PYTHONPATH=bindings/python/src python -m motorbridge --help
```

Closed-handle guard smoke:

```bash
PYTHONPATH=bindings/python/src python - <<'PY'
from motorbridge import Motor
m = Motor(0)
try:
    m.get_state()
except Exception as e:
    print(type(e).__name__, e)
PY
```

Expected: `CallError motor handle is closed`.

## RobStride Manual Checks

Parameter write/readback:

```bash
cargo run -p motor_cli -- --vendor robstride --channel can0 --model rs-00 --motor-id 1 --feedback-id 0xFD --mode read-param --param-id 0x7017
cargo run -p motor_cli -- --vendor robstride --channel can0 --model rs-00 --motor-id 1 --feedback-id 0xFD --mode write-param --param-id 0x7017 --param-value 0.9 --store 1
cargo run -p motor_cli -- --vendor robstride --channel can0 --model rs-00 --motor-id 1 --feedback-id 0xFD --mode read-param --param-id 0x7017
```
